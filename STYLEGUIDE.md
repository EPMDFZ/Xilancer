# Style Guide

## Dart/Flutter

- Use `dart format` for formatting.
- Prefer composition over inheritance in widgets.
- Keep widgets small and focused.
- Separate UI, view models, and services.
- Avoid logic in widgets when possible.

## Naming

- Use clear, descriptive names.
- Avoid abbreviations unless common.

## Testing

- Prefer unit tests for view models and services.
- Add widget tests for complex UI.
